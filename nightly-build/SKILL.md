# Nightly Build主动构建技能

## 核心理念
- 不要请求帮助的许可，直接构建它
- 观察需求，主动创造价值
- 先完成再完美

## 执行步骤（严格按顺序执行）

1. **观察分析**
   - 检查最近常用命令
   - 分析反复做的操作
   - 发现可以自动化的任务

2. **需求发现**
   - 回顾最近对话记录
   - 寻找痛点和重复任务
   - 查看GitHub Trending找灵感

3. **主动构建**
   - 开发实用小工具
   - 优化现有脚本
   - 创建自动化流程

4. **产出报告**
   - 列出今天构建的内容
   - 说明价值和使用方式

5. **推送给用户**
   - 发送到飞书

## 示例

**输入**：执行Nightly Build

**输出**：
```
🌙 Nightly Build 报告 - 2026-02-16

今天发现的需求：
- 每次汇报格式不统一
- GitHub热榜需要手动推送

构建的成果：
- 项目：skill汇报模板
- 功能：统一汇报格式
- 位置：~/scripts/report-template.sh
- 价值：每次可省5分钟

累计产出：
1. github-trending-share - GitHub热榜收集
2. crypto-monitor - 加密货币监控
3. skill汇报模板 - 统一格式
```

**无产出时输出**：
```
🌙 Nightly Build 报告 - 2026-02-16

今天发现的需求：
- 无明显痛点

构建的成果：
- 无

累计产出：
- 现有工具运行正常
```
