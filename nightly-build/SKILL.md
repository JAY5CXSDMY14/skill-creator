# Nightly Build主动构建技能

## 核心理念
- 不要请求帮助的许可，直接构建它
- 观察需求，主动创造价值
- 先完成再完美

## 执行步骤（严格按顺序执行）

1. **观察分析**
   - 检查最近常用命令
   - 分析反复做的操作
   - 发现可以自动化的任务

2. **需求发现**
   - 回顾最近对话记录
   - 寻找痛点和重复任务
   - 查看GitHub Trending找灵感

3. **主动构建**
   - 开发实用小工具
   - 优化现有脚本
   - 创建自动化流程

4. **产出报告**
   - 列出今天构建的内容
   - 说明价值和使用方式

5. **推送给用户**
   - 发送到飞书

## 输出格式

```
🌙 Nightly Build 报告 - 2026-02-16

今天发现的需求：
- 痛点1
- 痛点2

构建的成果：
- 项目名：xxx
- 功能：xxx
- 位置：~/xxx

累计产出：
- 工具1
- 工具2
```
